# 프로젝트 검색 / 정렬 / 페이징 기능 설계

## 목적
- 랜딩 페이지의 프로젝트 목록 조건 조회를 백엔드에서 책임진다.

## 지원 조건
- `type`: 전체 / 도급 / 상주
- `sort`: 기본 / 최신 / 금액 높은순 / 금액 낮은순 / 마감 임박순
- `page`
- `size=4`

## 미지원 조건
- 지역검색
- 참여파트 분류
- 프론트 단 sort

## Repository 전략
- `ProjectRepositoryCustom`에서 동적 조회 구현
- 조건별 where/order/pageable 조합

## 성능 메모
- 랜딩 카드에 필요한 최소 컬럼만 projection 조회
- 지원자 수는 `projects.applicant_count` 사용
- D-day는 조회 시점 계산

## 응답 모델 메모
- 금액 표기는 숫자와 표시 문구를 분리하지 않고 `budgetLabel`로 넘겨도 무방
- 프론트는 문자열 표시만 담당하고 의미 계산은 서버에서 한다

## 테스트
- 총 10건 이상 더미데이터 기준 페이징 검증
- 정렬 값별 결과 순서 검증
- 타입 필터 적용 검증
