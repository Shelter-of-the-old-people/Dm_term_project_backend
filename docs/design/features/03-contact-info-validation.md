# 지원 내용 연락처 차단 기능 설계

## 목적
- 지원서 내용에 직접 연락처를 적는 행위를 차단한다.

## 적용 대상
- 도급 지원서 `proposalText`
- 상주 지원서 `proposalText`

## 구현 위치
- 1차: DTO 검증 이후 서비스 계층에서 검사
- 2차: 공통 유틸 `ContactInfoValidator` 사용

## 탐지 대상
- 이메일 패턴
- 전화번호 패턴

## 실패 처리
- 에러 코드: `INVALID_INPUT`
- 메시지 예시: `지원 내용에는 이메일 또는 전화번호를 입력할 수 없습니다.`

## 확장 가능성
- 카카오톡 ID, 오픈채팅 링크, URL 탐지도 추후 추가 가능
- 현재 평가는 이메일/전화번호 차단만 충족하면 된다

## 테스트 케이스
- 일반 소개글 통과
- `010-1234-5678` 포함 시 실패
- `abc@test.com` 포함 시 실패
